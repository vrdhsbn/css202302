Gridレイアウト

カード風のコンポーネントをグリッド状に並べたりするときに便利。「お知らせ一覧」みたいなレイアウトはflexboxより簡単にできるしレスポンシブ対応も楽ちん。

.grid-container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 24px;
}

スマホ表示の時は2カラムにしたい…というときも以下のように grid-template-columns のrepeatを3から2に変更するだけでOK。また、いずれの場合でもグリッドコンテナの子要素である各アイテムの横幅は、いちいち指定せずともコンテナの3等分（または2等分）にしてくれる。

@media screen and (max-width: 767px) {
    .grid-container {
        grid-template-columns: repeat(2, 1fr);
    }
}

JS無しでのスムーススクロール

本ページ内のページ内リンクをクリックしたときのスムーススクロールにはJavaScriptを使っておらず、以下のCSSのみで実現している。

html {
    scroll-behavior: smooth;
}

このときのスクロール速度は変更できず、ブラウザに依存する。Safariでは素早くスクロールし、Chromeでは比較的ゆっくりスクロールする。

参考：https://developer.mozilla.org/ja/docs/Web/CSS/scroll-behavior

論理プロパティ

従来は margin-right や top のように「上下左右」の値でマージンやパディング、位置を指定していた（物理プロパティ）。物理プロパティの代わりに、論理プロパティである inline / block や start / end を使った書き方ができる。

以下のようなプロフィールカードをレイアウトするとき、アバター画像とプロフィールテキストとの間にマージンを開けたいとする。このとき、margin-left ではなく margin-inline-start を使って書くことができる。

.profile-detail {
    margin-inline-start: 24px;
}

普段はあまり意識しないかもしれないが、アラビア語やヘブライ語のような右から左に向かう書字方向を使うサイトや、縦書きを用いるサイトでは活用できるシーンがあると思われる。

便利な擬似クラス

:is() や :where() でセレクタを一括指定できる。

以下のように書くと .link1 と .link2 配下のリンクにホバーしたときのみスタイルを適用できる。

:is(.link1, .link2) a:hover {
    opacity: .6;
}

link2

link3

:where() も同じ構文で同じことができるが、:is() との違いは詳細度を持つか持たないか。:where() の詳細度はつねに0であり、:is() の詳細度は引数のセレクタに応じて決まる。:is(.link1, .link2) の例だとクラスセレクタが2つあるのでこの詳細度は (0,2,0) になる。

参考：https://developer.mozilla.org/ja/docs/Web/CSS/:is
